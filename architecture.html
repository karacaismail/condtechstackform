Request hayat döngüsü — katmanlar boyunca

Eğitim için: bir HTTP isteği aşağıdaki sırayla katmanlardan geçer. Her geçişte hangi sorumluluğun yerine getirildiğini ekibinize açıklayabilirsiniz.

1. Client

Kullanıcı PWA'da form gönderir. Service Worker offline ise IndexedDB'ye kuyruğa alır, sonra retry.

2. Edge

Caddy TLS terminate. Write Protection JWT + tenant + CSRF doğrular, rate limit kontrol, security header ekler.

3. API

FastAPI route Pydantic ile validate. OpenAPI'den üretilen tip + RPC SDK aynı kontrak.

4. Application

Vertical slice handler use-case'i çağırır; gerekirse Agent Runtime tetiklenir (AI tool-call) veya ECA Engine event yayınlar.

5. Domain

Bounded context içinde iş kuralı uygulanır. Field/Custom Field API doğrulaması yapılır.

6. AI

Agent gerekirse Provider Abstraction üzerinden LLM çağırır; Hybrid Ranker bağlam getirir; tüm çağrı AI Audit'e düşer.

7. Tenant

Tenant Resolver tenant_id'yi enjekte eder; Quota Gate plan limitini kontrol eder; brand theme UI'a yayılır.

8. Persistence

Postgres + Prisma ile yazma; gerekirse Elastic/pgvector reindex event'i Redis pub/sub'a düşer.

9. DX

CLI/Designer üretilen kod buradan akar. Geliştirme aşamasında — runtime'da yer almaz, geliştirici loop'unun parçası.

10. Ops

Tüm adımlar log + metric + trace olarak yayılır. Audit Trail yazma işlemlerini immutable kayıtla saklar.

Anti-pattern uyarıları

Eğitim oturumunda katman ihlali olarak işaretleyin — bunlar gözden kaçırılınca framework çöker.

Cross-layer leak

Domain bounded context bir başkasını doğrudan import etmesin — sadece event veya port üzerinden.

Tenant_id atlama

Sorgu yazarken WHERE'a tenant_id koymayı unutma. Middleware'den otomatik ekleyen helper kullan, manuel SQL yazma.

Plugin core'a sızıntı

Plugin core entity'ye doğrudan dokunamaz; sadece ilan ettiği hook + API üzerinden.

Senkron AI çağrısı

LLM çağrısı request handler içinde bloke etmesin — agent runtime queue'ya at, SSE ile sonucu döndür.

Schema değişikliği commit'siz

Doctype Designer ile yapılan değişiklik git'e commit'lenmeden tenant tarafına yansımasın.

Audit log es geçme

Yazma operasyonları audit'siz kabul edilmesin. Yasal compliance + güven için zorunlu.
